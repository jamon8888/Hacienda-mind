Three ways to run hacienda-mcp, easiest first. All three share the same local index
and are safe to run side by side.

<Aside type="tip">
  The plugin downloads the hacienda-mcp program for you on first use. The MCP
  server and CLI paths need it installed yourself — see [Install the
  program](#install-the-program).
</Aside>

## 1. As a plugin (recommended)

The plugin sets up everything for you — the server, the helper skills, the
agent-comms features, and the slash commands. Pick your coding tool.

<Tabs syncKey="plugin">
  <TabItem label="Claude Code">
    In the session (not your shell), run in order:

    <Code code={`/plugin marketplace add jamon8888/Hacienda-mind\n/plugin install hacienda-mcp@hacienda-mcp`} lang="text" />

    Restart, then run `/bm-statusline` once to turn on the live statusline (a
    one-time step — see [Statusline](#statusline-claude-code)). Recommended:
    turn on auto-update for the `hacienda-mcp` marketplace in Claude Code's plugin
    manager, so you always get the current index format and tool set — or
    update it regularly by hand if you'd rather control timing.

  </TabItem>

  <TabItem label="Codex">
    <Code code={`codex plugin marketplace add jamon8888/Hacienda-mind\ncodex plugin add hacienda-mcp@hacienda-mcp`} lang="bash" />

    In the app: open the **Plugins** sidebar and add hacienda-mcp. The CLI and IDE
    share one config file.

  </TabItem>

<TabItem label="Cursor">
  In Agent chat: `/add-plugin hacienda-mcp` (once listed), or go to **Dashboard
  → Settings → Plugins → Team Marketplaces → Import from Repo** and point it at
  `https://github.com/jamon8888/Hacienda-mind`.
</TabItem>

  <TabItem label="Gemini CLI">
    <Code code={`gemini extensions install https://github.com/jamon8888/Hacienda-mind`} lang="bash" />

    Update later with `gemini extensions update hacienda-mcp`.

  </TabItem>

<TabItem label="Factory Droid">
  <Code
    code={`droid plugin marketplace add https://github.com/jamon8888/Hacienda-mind\ndroid plugin install hacienda-mcp@hacienda-mcp`}
    lang="bash"
  />
</TabItem>

<TabItem label="GitHub Copilot CLI">
  <Code
    code={`copilot plugin marketplace add jamon8888/Hacienda-mind\ncopilot plugin install hacienda-mcp@hacienda-mcp`}
    lang="bash"
  />
</TabItem>

  <TabItem label="OpenCode">
    Add to `opencode.json` (project) or `~/.config/opencode/opencode.json`
    (global):

    <Code code={`{ "plugin": ["basemind-opencode@latest"] }`} lang="json" />

  </TabItem>

  <TabItem label="Kimi Code">
    <Code code={`/plugins install https://github.com/jamon8888/Hacienda-mind`} lang="text" />

    Kimi doesn't support the comms auto-notifications, but the chat tools still
    work.

  </TabItem>

  <TabItem label="Hermes">
    <Code code={`pip install hacienda-mcp`} lang="bash" />

    That ships a native Hermes plugin (the helper skills, slash commands, and
    comms notifications), auto-discovered via the `hermes_agent.plugins` entry
    point. Hermes wires tools through config, so add hacienda-mcp's tools with the
    [MCP block](#2-as-an-mcp-server) (`mcp_servers.hacienda-mcp` in
    `~/.hermes/config.yaml`), then enable the plugin:

    <Code code={`hermes plugins enable hacienda-mcp`} lang="bash" />

    Comms auto-notifications are best-effort; the chat tools work regardless.

  </TabItem>

  <TabItem label="Antigravity & pi">
    **Antigravity** uses a shared MCP config — [install the program](#install-the-program),
    then add the [generic MCP block](#2-as-an-mcp-server). If you already use the
    Gemini extension, `agy plugin import gemini` brings it across.

    **pi**: `pi install git:github.com/jamon8888/Hacienda-mind`. pi has no MCP support,
    so hacienda-mcp runs through its [CLI](#3-as-a-cli) here.

  </TabItem>
</Tabs>

## 2. As an MCP server

If your tool speaks MCP but you're not using the plugin,
[install the program](#install-the-program), then register it:

<Code
  code={`{
  "mcpServers": {
    "hacienda-mcp": { "command": "hacienda-mcp", "args": ["serve"] }
  }
}`}
  lang="json"
/>

Each tool says whether it only reads or can change things, so your client can
auto-approve the safe ones and ask before the rest. If `hacienda-mcp` isn't found,
use the full path from `which hacienda-mcp`.

## 3. As a CLI

The standalone program, for scripts, headless runs, and CI.
[Install it](#install-the-program), then:

<Code
  code={`hacienda-mcp scan                          # index the project once
hacienda-mcp query symbol "parseQuery"     # find a symbol by name
hacienda-mcp query references "processFile" # find everywhere it's called
hacienda-mcp git blame-file src/main.rs    # who last changed each line
hacienda-mcp watch                         # keep the index fresh as files change`}
  lang="bash"
/>

Full command list in the [CLI reference](/reference/cli/).

## Install the program

The MCP and CLI paths need `hacienda-mcp` available on your system. (The plugin does
this for you.)

| Channel         | Command                                                                  | Includes        |
| --------------- | ------------------------------------------------------------------------ | --------------- |
| Homebrew        | `brew install Goldziher/tap/hacienda-mcp`                                | everything      |
| npm             | `npm install -g hacienda-mcp`                                            | everything      |
| pip             | `pip install hacienda-mcp`                                               | everything      |
| cargo           | `cargo install hacienda-mcp --locked`                                    | code + git only |
| cargo (full)    | `cargo install hacienda-mcp --features full --locked`                    | everything      |
| GitHub releases | [Download a binary](https://github.com/jamon8888/Hacienda-mind/releases) | everything      |

The Homebrew / npm / pip / GitHub downloads include the full feature set —
documents, OCR, search, web crawl, shared memory, agent comms, and agent shells
— so the first run downloads the models it needs. The plain `cargo install`
builds the code-map and git tools only.

## Get started

After installing, run **`hacienda-mcp init`** (CLI) — or **`/bm-init`** if your
tool supports slash commands — from the repo root. It's re-runnable and safe
to call again later:

- Writes a commented `basemind.toml` scaffold at the repo root, if one doesn't
  already exist.
- Lets you pick which capabilities to advertise (interactive prompt in a TTY,
  or non-interactive with `--yes`, `--with <capability>`,
  `--without <capability>`). Capability slugs: `code-search-navigation`,
  `code-mapping-architecture`, `git-history`, `agent-comms`, `documents-rag`,
  `semantic-search`.
- Injects a "prefer hacienda-mcp over grep/read/git" rules block into your repo's
  agent-instructions file — `.ai-rulez/rules/basemind-usage.md` if
  `.ai-rulez/config.toml` is present (run `ai-rulez generate` afterward), else
  `CLAUDE.md`, else `AGENTS.md`, else a new `CLAUDE.md`. The block is delimited
  (`<!-- BEGIN hacienda-mcp ... -->` / `<!-- END hacienda-mcp -->`) so re-running
  replaces it in place instead of duplicating it.

Preview changes without writing with `--print`; skip the rules step with
`--no-rules`; steer the target explicitly with
`--rules-target <auto|claude|agents|ai-rulez|none>`.

### Statusline (Claude Code)

Run `/bm-statusline` once. This is a one-time step because Claude Code doesn't
let plugins set the main statusline themselves — so hacienda-mcp asks the assistant
to make the one-line settings change on your behalf, and it sticks from then on.

It shows two lines:

```text
Opus · hacienda-mcp · ⎇ main · 12% ctx
◆ hacienda-mcp  ●  1,247 files · 23m ago  │  312 calls · 180 srch · 44 git · 12 docs  │  1.4M saved  │  ✉ 3 @reviewer
```

The dot is green when hacienda-mcp is live and fresh, amber when idle, red when
stale. The middle shows activity by type, then tokens saved, then unread
messages. Adjust with `HACIENDA_MCP_STATUSLINE=full|compact|minimal`, or hide the
top line with `HACIENDA_MCP_STATUSLINE_CONTEXT=0`.
